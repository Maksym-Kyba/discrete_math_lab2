Звіт

Мета:

Реалізувати клас, який отримує регулярний вираз на вхід, компілює для нього скінченний автомат і використовує його для перевірки рядків.


Виконано


Реалізовано всі класи станів (StartState, TerminationState, DotState, AsciiState, StarState, PlusState) на основі патерну проектування State;

Реалізовано побудову графа автомата з regex-рядка в RegexFSM.__init__;

Реалізовано симуляцію НКА методом підмножин у check_string з підтримкою е-стрілок через _e_expand;

Виправлено баг із спільними next_states на рівні класу (всі списки перенесено в __init__ через super().__init__()).

Примітка: Код підтримує англійські літери, цифри та будь-який одиночний символ.


Висновок: патерн State дозволяє гарно розділити логіку кожного символа по окремих класах. Метод підмножин із е-стрілками дає змогу правильно відтворити недетермінований автомат без перетворення його в детермінований.



Пояснення імплементації
1. State Патерн
Кожен вузол автомата - окремий підклас абстрактного класу State. Кожен клас несе відповідальність за один конкретний тип символа:

StartState відповідає за початковий вузол і нічого не приймає

TerminationState - це кінцевий стан. Якщо він є отже рядок оброблений

AsciiState відповідає за один символ (літеру або цифру)

DotState відповідає це будь-який одинарний символ

PlusState та StarState відповідають за повторення (StarState від нуля а PlusState від одного)



2. Побудова графа - RegexFSM
Кожен символ regex обробляється по черзі й породжує новий вузол.

*: Метод _locate_parent обходить граф у ширину (BFS) і знаходить батьківський стан для попереднього символа. Ребро parent → попередній_стан замінюється на parent → StarState, а до StarState додається петля для багаторазового повторення.

+: Попередній стан залишається в ланцюзі (це гарантує хоча б одне повторення). PlusState додається після нього також з петлею.



3. Симуляція рядка - check_string
Використовується метод підмножин:

Початкова активна множина — прямі наступники StartState, після чого виконується замикання.

Епсилон-замикання розкриває нуль-матч шляхи: якщо StarState або PlusState активні, наступні стани (крім петлі) одразу додаються до множини без споживання символу.

Для кожного символу вхідного рядка: кожен активний стан, де check_self повертає True, переводить автомат у свої наступники; петльові стани також залишаються активними.

Після обробки всього рядка перевіряється наявність TerminationState в активній множині.
